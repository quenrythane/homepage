Lekcja 04.02 Jest GIT

- Git to jak **check pointy**- po dojściu do jakiegoś punktu robisz zapis. I od teraz, jeśli w dalszej pracy **coś zjebiesz** to zawsze **możesz wrócić** do checkpointu

- Git składa się z **repozytorium = nasz katalog = nasz projek**

- Co to jest branch master?

- Git komendy:
- git --version
- git config --global user.name "Artur Babiński"
- git config --global user.email "artur.babinski.g@gmail.com"
- git config --get user.name
- git config --get user.email
- git init- inicjulizuje puste repozytorium w katalogu homepage
powstaje nowy plik .git, który jest plikiem ukrytym i trzyma całą historię zmian w sobie
- commit = **stan projektu** i opis tego co się zmieniło od ostatniego zapisu
np. powstał nowy plik o nazwie "nazwa"
wprowadzono zmiany w pliku o nazwie "index"
- **git status**- wypisuje status naszego repozytorium- po prostu informacje o naszym repozytorium.
Wymienia też pliki które są w katalogu ale nie są śledzone (tzn, że nie robi odnośnie nich żdanych czynności, w tym nie robi check pointów
- **git add .** - dodaje wszystkie pliki które są w katalogu w którym się znajdujemy (jak i w katalogach niżej) do repozytorium (tzn, że od teraz je śledzi)
- **git commit -m "message/description"** -Stworzy nowy commit z wiadomością (-m jak message) którą zapisaliśmy w cudzysłowie. Dobrą praktyką jest tą wiadomość pisać po angielsku.
Dodatkowo zostanie wygenerowany unikalny #hash danego commita
- gitk -pokazuje całą historie repozytorium

- Jak często commitować?
- najlepiej bardzo często- im więcej tym lepiej
- przed commitem zawsze sprawdź, czy twój kod się odpala i działa tak jak chcesz

- git /git lens
Mamy też takiego gita zainstalowanego w wizualnej wersji w naszym edytorze kodu
Dzięki temu z poziomu edytora kodu także możemy podglądać co się zmieniało w naszym kodzie
- git lens jest dodatkiem do visual studio

04.03 GitHub

- Jest to twoja programistyczna wizytówka

- Służy do dzielenia się twoim projektem i dawaniem dostępu do kodu ale bez dawania dostępu do twojego dysku/komputera

- Tworząc repozytorium dobrą praktyką jest to, by miało taką samą nazwę jak twój katalog z projektem

- kopiujemy te linkijki do terminala i klikamy enter:

```
git remote add origin https://github.com/quenrythane/homepage.git
git branch -M master
git push -u origin master
```

- Jeśli na danym urządzeniu jeszcze się nie logowaliśmy to będziemy w terminalu musieli zalogować się do githuba
- wpisując na początku nazwę użytkownika i klikając enter
- następnie podaj hasło (uwaga! hasło nie będzie się wyświetlać- kiedy będziemy je wpisywać będzie wyglądać jakby nic się nie wpisywało
